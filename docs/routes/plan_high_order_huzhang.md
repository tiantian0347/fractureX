# 规划：Hu–Zhang 高次元在论文中的定位

> 状态：草案 v0.1（2026-05-30）。本文件记录"是否/如何在论文中体现 Hu–Zhang 高次元"的
> 决策，供 GPU+多后端、算子学习两条论文路线取用。
> 相关：[plan_gpu_multibackend.md](plan_gpu_multibackend.md)、
> [plan_operator_learning.md](../operator_learning/plan_operator_learning.md)、
> 自适应延伸见本文件 §4。

## 0. 一句话结论

**要体现 Hu–Zhang 高次，但放在「光滑弹性验证 + 等 DOF 应力精度对比」里，
不要放在「带裂纹算例的收敛率」上。** 高次是手段（拿到准 σ 去喂相场驱动场），
不是要论证的结论。

## 1. 先分清两层

- *能算高次*：工程能力。fracturex 中标准有限元与 Hu–Zhang 都支持 p≥1。
- *高次值得作为论文论证点*：是否能支撑论文的**核心论点**。

两者不能混。**能算 ≠ 值得论证**。

## 2. Hu–Zhang 高次的真实价值与边界

**价值。** Hu–Zhang 的卖点是 $H(\mathrm{div};\mathbb S)$ 协调 + 应力逐点对称，
高次时应力 $\sigma$ 能取到最优阶 $O(h^{k+1})$。这对相场断裂有内在意义：驱动场 $H$
（正应变能密度 $\psi^+$）本质上吃应力/应变，$\sigma$ 越准，裂纹驱动力和裂纹路径越可信。
这也正是 σ 通道走直接求值、默认 p=3 的逻辑链条
（见 [SURROGATE_DATA_SCHEMA.md](../operator_learning/SURROGATE_DATA_SCHEMA.md) 与代码侧 `_evaluate_huzhang_on_grid`）。

**边界（绕不开的反作用）。** 相场解在裂纹带附近**正则性很差**（d 局部化、σ 梯度陡）。
在真正开裂的算例上做 p-refinement，**看不到干净的高阶收敛率**——裂尖奇异会把阶压回去。
不要把带裂纹域上的高阶收敛*率*当卖点，否则会被反问"裂尖奇异下额外 DOF 值不值"。

## 3. 推荐分工（不是一锅炖）

1. **验证层**：在**光滑弹性问题**（MMS / patch test，无损伤 d=0）放**一个**
   p-refinement 收敛研究，展示 Hu–Zhang p=2,3 对 σ 取到最优阶。目的：证明实现正确、
   立 mixed 方法的信誉——干净、无争议的图。单元测试地基见
   `fracturex/tests/test_recover_strain.py`（σ→ε 往返解析校验）。
2. **生产层**：实际断裂算例用**适中阶**（p=2 或 p=3）跑；主论点放
   **标准 FEM vs Hu–Zhang 在等 DOF 下的应力精度对比**，而非"高次本身有多好"。
   等 DOF 对比比 p-扫描更有说服力、更难被攻击。
3. **不要**在裂纹算例上把高阶收敛率当卖点。

## 4. 自适应延伸（非光滑高次的正确出口）

非光滑场上"追精度"的正确做法不是全局升 p / 升 h，而是
**a posteriori 误差估计驱动的自适应加密**，把 DOF 堆到裂纹带/裂尖。

候选方向（待展开）：
- 与 Hu–Zhang 混合格式天然匹配的 a posteriori 估计子（σ 是主变量 ⇒ 残差/平衡型估计子直接）。
- hp-自适应：光滑远场用高 p，裂纹带内 h-加密（或 p-降阶）。
- 标记/加密与相场损伤指示量 d 耦合。

这是 backlog 里"自适应 + a posteriori"那条线的一篇独立工作量，本规划只占其入口。

## 5. 按论文路线的权重

两条线对"高次"的需求权重不同：

- **算子学习路线**更在乎 σ 数据质量 ⇒ 高次理由更强，建议给完整验证层 + 等 DOF 对比。
- **GPU+多后端路线**更偏能力/验证项 ⇒ 高次作为正确性与覆盖度的支撑即可。

各路线据此决定该节篇幅与放置的 §。
